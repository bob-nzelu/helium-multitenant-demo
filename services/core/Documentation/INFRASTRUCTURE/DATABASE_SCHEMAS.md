**Version:** 1.0
**Date:** 2026-02-05
**Phase:** Phase 0a (Infrastructure Architecture)
**Status:** REFERENCE DOCUMENT - Complete SQL schemas for all Core databases

## OVERVIEW

This document contains complete SQL schemas for all four Core databases:

1. **invoices.db** (production) - PostgreSQL
2. **invoices_test.db** (test environment) - SQLite
3. **customers.db** - PostgreSQL (shared test/production with environment flags)
4. **inventory.db** - PostgreSQL (shared test/production with environment flags)

**Key Decisions**:
- PostgreSQL for production (better concurrency, scalability)
- SQLite for test invoices (fast local development, complete isolation)
- Shared customers/inventory (master data is company-wide, not environment-specific)
- Test/production isolation via `environment` field ("test" vs "production")

## 1. INVOICES.DB (PRODUCTION) - PostgreSQL

**Purpose**: System of record for invoice lifecycle in production environment

**Database Type**: PostgreSQL 14+

**Schema Version**: 1.0

### Table: invoices

```sql
-- Main invoice records (production only)
CREATE TABLE invoices (
    -- Primary identification
    id SERIAL PRIMARY KEY,
    invoice_id VARCHAR(100) UNIQUE NOT NULL,     -- User-visible invoice ID (from raw data)
    irn VARCHAR(64) UNIQUE,                       -- Invoice Reference Number (hash-based, generated by Core)

    -- Queue tracking
    queue_id VARCHAR(50) UNIQUE,                  -- core_queue ID (for tracking)
    batch_id VARCHAR(50),                         -- Batch ID (for bulk uploads)

    -- Blob storage reference
    blob_uuid VARCHAR(50) NOT NULL,               -- File UUID in blob storage
    original_filename VARCHAR(255),               -- Original uploaded filename

    -- Invoice metadata
    invoice_date DATE,                            -- Invoice date (from raw data)
    due_date DATE,                                -- Payment due date
    currency VARCHAR(3) DEFAULT 'NGN',            -- Currency code (ISO 4217)

    -- Customer reference
    customer_id INTEGER REFERENCES customers(id) ON DELETE SET NULL,
    customer_name VARCHAR(255),                   -- Denormalized for fast access
    customer_tin VARCHAR(20),                     -- Tax Identification Number
    customer_rc_number VARCHAR(20),               -- RC Number

    -- Financial totals
    subtotal NUMERIC(15, 2),                      -- Subtotal before tax
    tax_amount NUMERIC(15, 2),                    -- Total tax amount
    total_amount NUMERIC(15, 2) NOT NULL,         -- Total invoice amount

    -- FIRS submission
    firs_status VARCHAR(50) DEFAULT 'DRAFT',      -- 'DRAFT', 'SUBMITTED', 'VALIDATED', 'REJECTED'
    firs_confirmation VARCHAR(100),               -- FIRS confirmation number
    firs_submitted_at TIMESTAMP,                  -- When submitted to FIRS
    firs_response_data JSONB,                     -- Complete FIRS response

    -- QR code
    qr_code_data TEXT,                            -- QR code content (Base64 encoded image)

    -- Company context
    company_id VARCHAR(50) NOT NULL,              -- Company identifier

    -- Lifecycle tracking
    status VARCHAR(50) DEFAULT 'DRAFT',           -- 'DRAFT', 'PREVIEW_READY', 'FINALIZED', 'SUBMITTED', 'PAID', 'ARCHIVED'
    created_by VARCHAR(255),                      -- User email who created/uploaded
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    finalized_at TIMESTAMP,                       -- When user confirmed preview

    -- Audit trail
    processing_started_at TIMESTAMP,              -- Core processing start time
    processing_completed_at TIMESTAMP,            -- Core processing end time
    processing_duration_ms INTEGER,               -- Processing time in milliseconds

    -- Soft delete
    deleted_at TIMESTAMP,                         -- Soft delete (24-hour recovery window)
    deleted_by VARCHAR(255),                      -- User who deleted

    -- Indexes
    INDEX idx_invoice_id ON invoices(invoice_id),
    INDEX idx_irn ON invoices(irn),
    INDEX idx_queue_id ON invoices(queue_id),
    INDEX idx_batch_id ON invoices(batch_id),
    INDEX idx_customer_id ON invoices(customer_id),
    INDEX idx_firs_status ON invoices(firs_status),
    INDEX idx_status ON invoices(status),
    INDEX idx_company_id ON invoices(company_id),
    INDEX idx_created_at ON invoices(created_at),
    INDEX idx_invoice_date ON invoices(invoice_date)
);

-- Trigger: Update updated_at on row modification
CREATE TRIGGER update_invoices_updated_at
BEFORE UPDATE ON invoices
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();

-- Function for updated_at trigger
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- WebSocket broadcast trigger
CREATE TRIGGER invoice_created_trigger
```

### Table: invoice_line_items

```sql
-- Line items for each invoice
CREATE TABLE invoice_line_items (
    id SERIAL PRIMARY KEY,

    -- Parent invoice
    invoice_id INTEGER REFERENCES invoices(id) ON DELETE CASCADE,

    -- Line item details
    line_number INTEGER NOT NULL,                 -- Line number (1, 2, 3...)
    description TEXT,                             -- Item description
    quantity NUMERIC(15, 4) NOT NULL,             -- Quantity
    unit_price NUMERIC(15, 2) NOT NULL,           -- Price per unit
    line_total NUMERIC(15, 2) NOT NULL,           -- quantity * unit_price

    -- Tax details
    tax_rate NUMERIC(5, 2),                       -- Tax rate (e.g., 7.5 for 7.5%)
    tax_amount NUMERIC(15, 2),                    -- Tax for this line

    -- Product reference
    product_id INTEGER REFERENCES inventory(id) ON DELETE SET NULL,
    product_code VARCHAR(100),                    -- SKU or product code
    product_name VARCHAR(255),                    -- Denormalized product name

    -- HSN code (for FIRS compliance)
    hsn_code VARCHAR(20),                         -- Harmonized System of Nomenclature

    -- Audit
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- Indexes
    INDEX idx_invoice_id ON invoice_line_items(invoice_id),
    INDEX idx_product_id ON invoice_line_items(product_id),
    UNIQUE INDEX idx_invoice_line_unique ON invoice_line_items(invoice_id, line_number)
);
```

**Database Type**: PostgreSQL 14+

**Schema Version**: 1.0

**Note**: This database was mentioned in CORE_ARCHITECTURE.md but not fully specified

### Table: notifications

```sql
-- System notifications
CREATE TABLE notifications (
    id SERIAL PRIMARY KEY,

    -- Notification details
    notification_type VARCHAR(50) NOT NULL,       -- 'INFO', 'WARNING', 'ERROR', 'APPROVAL_REQUEST'
    title VARCHAR(255) NOT NULL,
    message TEXT NOT NULL,

    -- Target
    recipient_email VARCHAR(255),                 -- Target user (NULL for broadcast)
    recipient_role VARCHAR(50),                   -- Or target role (e.g., 'ADMIN', 'APPROVER')

    -- Context
    related_entity_type VARCHAR(50),              -- 'INVOICE', 'CUSTOMER', 'PRODUCT'
    related_entity_id VARCHAR(100),               -- ID of related entity

    -- Company context
    company_id VARCHAR(50) NOT NULL,

    -- Lifecycle
    status VARCHAR(50) DEFAULT 'UNREAD',          -- 'UNREAD', 'READ', 'DISMISSED'
    read_at TIMESTAMP,
    dismissed_at TIMESTAMP,

    -- Audit
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- Indexes
    INDEX idx_recipient_email ON notifications(recipient_email),
    INDEX idx_recipient_role ON notifications(recipient_role),
    INDEX idx_status ON notifications(status),
    INDEX idx_notification_type ON notifications(notification_type),
    INDEX idx_company_id ON notifications(company_id),
    INDEX idx_created_at ON notifications(created_at)
);

-- WebSocket broadcast trigger
CREATE TRIGGER notification_created_trigger
AFTER INSERT ON notifications
FOR EACH ROW
EXECUTE FUNCTION broadcast_notification_created();

CREATE TRIGGER notification_updated_trigger
AFTER UPDATE ON notifications
FOR EACH ROW
EXECUTE FUNCTION broadcast_notification_updated();

-- Broadcast functions
CREATE OR REPLACE FUNCTION broadcast_notification_created()
RETURNS TRIGGER AS $$
BEGIN
    PERFORM pg_notify('notification_events', json_build_object(
        'event_type', 'notification.created',
        'notification_id', NEW.id,
        'data', row_to_json(NEW)
    )::text);
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE OR REPLACE FUNCTION broadcast_notification_updated()
RETURNS TRIGGER AS $$
BEGIN
    PERFORM pg_notify('notification_events', json_build_object(
        'event_type', 'notification.updated',
        'notification_id', NEW.id,
        'data', row_to_json(NEW)
    )::text);
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

## 6. CORE_QUEUE - Part of core.db (Not Standalone)

**Purpose**: Queue for Core processing (separate from invoices.db)

**Database Type**: PostgreSQL 14+ (production), SQLite (test)

**Ownership**: Core service (written by Relay, read by Core)

### Table: core_queue (PostgreSQL)

```sql
-- Queue entries for Core processing
CREATE TABLE core_queue (
    id SERIAL PRIMARY KEY,

    -- Queue identification
    queue_id VARCHAR(50) UNIQUE NOT NULL,         -- Unique queue entry ID

    -- Blob reference
    blob_uuid VARCHAR(50) NOT NULL,               -- File UUID in blob storage
    original_filename VARCHAR(255),

    -- Processing mode
    immediate_processing BOOLEAN DEFAULT TRUE,    -- TRUE = auto-process, FALSE = preview mode

    -- Batch context
    batch_id VARCHAR(50),                         -- Batch ID (for bulk uploads)

    -- Company context
    company_id VARCHAR(50) NOT NULL,
    uploaded_by VARCHAR(255),

    -- Queue status
    status VARCHAR(50) DEFAULT 'PENDING',         -- 'PENDING', 'PROCESSING', 'COMPLETED', 'FAILED'
    processing_started_at TIMESTAMP,
    processing_completed_at TIMESTAMP,
    error_message TEXT,

    -- Priority
    priority INTEGER DEFAULT 5,                   -- 1 = highest, 10 = lowest

    -- Lifecycle
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- Indexes
    INDEX idx_queue_id ON core_queue(queue_id),
    INDEX idx_blob_uuid ON core_queue(blob_uuid),
    INDEX idx_batch_id ON core_queue(batch_id),
    INDEX idx_status ON core_queue(status),
    INDEX idx_priority_created ON core_queue(priority, created_at),
    INDEX idx_company_id ON core_queue(company_id)
);

-- Trigger: Update updated_at
CREATE TRIGGER update_core_queue_updated_at
BEFORE UPDATE ON core_queue
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();
```

## DATABASE INITIALIZATION SCRIPTS

### PostgreSQL Initialization (Production)

```bash
#!/bin/bash
# init_postgres_databases.sh

# Initialize all PostgreSQL databases for Core service

# Configuration
DB_HOST="localhost"
DB_PORT="5432"
DB_USER="helium_admin"
DB_PASSWORD="${HELIUM_DB_PASSWORD}"

# Create databases
psql -h $DB_HOST -p $DB_PORT -U $DB_USER <<EOF
CREATE DATABASE invoices;
CREATE DATABASE customers;
CREATE DATABASE inventory;
CREATE DATABASE notifications;
CREATE DATABASE core;
EOF

# Apply schemas
psql -h $DB_HOST -p $DB_PORT -U $DB_USER -d invoices < schemas/invoices.sql
psql -h $DB_HOST -p $DB_PORT -U $DB_USER -d customers < schemas/customers.sql
psql -h $DB_HOST -p $DB_PORT -U $DB_USER -d inventory < schemas/inventory.sql
psql -h $DB_HOST -p $DB_PORT -U $DB_USER -d notifications < schemas/notifications.sql
psql -h $DB_HOST -p $DB_PORT -U $DB_USER -d core < schemas/core_queue.sql

echo "PostgreSQL databases initialized successfully"
```

### SQLite Initialization (Test)

```python
#!/usr/bin/env python3
# init_sqlite_databases.py

import sqlite3
from pathlib import Path

# Initialize all SQLite databases for Test tier

DB_DIR = Path("./data/test")
DB_DIR.mkdir(parents=True, exist_ok=True)

def init_database(db_path: Path, schema_path: Path):
    """Initialize SQLite database with schema"""
    conn = sqlite3.connect(db_path)
    with open(schema_path) as f:
        conn.executescript(f.read())
    conn.commit()
    conn.close()
    print(f"Initialized: {db_path}")

# Initialize databases
init_database(DB_DIR / "invoices_test.db", Path("schemas/invoices_sqlite.sql"))
init_database(DB_DIR / "core.db", Path("schemas/core_queue_sqlite.sql"))

print("SQLite databases initialized successfully")
print("Note: customers.db and inventory.db use PostgreSQL even in Test tier")
```

## MIGRATION NOTES

### Test to Production Migration

When migrating from Test to Production:

1. **Invoices**: Do NOT migrate (test invoices are sandbox-only)
2. **Customers**: Migrate only `environment = 'production'` records
3. **Inventory**: Migrate only `environment = 'production'` records
4. **Notifications**: Do NOT migrate (test-specific)

```sql
-- Export production customers for migration
COPY (
    SELECT * FROM customers WHERE environment = 'production'
) TO '/tmp/customers_production.csv' CSV HEADER;

-- Import to production database
COPY customers FROM '/tmp/customers_production.csv' CSV HEADER;
```

## PERFORMANCE CONSIDERATIONS

### Index Strategy

- **Primary Keys**: All tables have auto-incrementing integer primary keys
- **Foreign Keys**: Indexed automatically by PostgreSQL
- **Lookup Fields**: TIN, RC Number, SKU, IRN all have indexes
- **Date Ranges**: created_at, invoice_date indexed for time-based queries
- **Status Fields**: Indexed for filtering (e.g., firs_status, status)

### Query Optimization Tips

```sql
-- Good: Use indexed fields
SELECT * FROM invoices WHERE firs_status = 'SUBMITTED';

-- Bad: Non-indexed function on field
SELECT * FROM invoices WHERE LOWER(customer_name) = 'acme';

-- Good: Use normalized field
SELECT * FROM customers WHERE company_name_normalized = 'acme corp';
```

### Partition Strategy (Future)

For Enterprise deployments with >1M invoices:

```sql
-- Partition invoices by year
CREATE TABLE invoices_2026 PARTITION OF invoices
    FOR VALUES FROM ('2026-01-01') TO ('2027-01-01');
```
